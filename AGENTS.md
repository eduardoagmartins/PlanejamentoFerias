## Spec Kit + Lean Inception

For new product features, run Lean Inception discovery before `/specify`.

1. Use `$lean-inception` to create a Spec Kit Discovery Handoff.
2. Store or reference the handoff as `specs/<feature>/discovery.md`.
3. Use `/specify` from the handoff, focusing on MVP1 only.
4. Use `/plan` with Lean Inception trade-offs, risks, constraints, and dependencies.
5. Use `/tasks` from the MVP1 sequencer and validation plan.
6. Do not promote brainstormed features into requirements unless they trace to a persona, journey, objective, MVP Canvas block, or stakeholder decision.
